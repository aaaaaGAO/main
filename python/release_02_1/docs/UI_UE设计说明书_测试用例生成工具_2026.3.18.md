2 Overall UI strategy  界面设计总则

The UI follows a four stage guided workflow per tab  每个业务页签采用四段式引导流程  
Stage 1 import testcase source  阶段一 导入用例源  
Stage 2 associate configuration tables  阶段二 关联配置表  
Stage 3 filter and select cases  阶段三 筛选与勾选用例  
Stage 4 output path and one click run  阶段四 输出路径与一键运行  

3 Global toolbar and common operations  全局工具栏与通用操作

Toolbar location  工具栏位置  top right of the page  页面右上角  
Save config preset  保存配置  save current UI state into a preset file  保存当前 UI 状态  
Import config preset  导入配置  load a preset file and restore UI state  导入配置并恢复 UI 状态  
Tabs  页签  LR_REAR CENTRAL DTC  
Status field  状态字段  each input shows selected or not selected  每个输入项显示已选择或未选择  

4 Core configuration files and persistence  核心配置文件与状态持久化

filter_options.txt  UI option source  UI 选项数据源  options for priority platform model target version  选项用于优先级 平台 车型 目标版本  
Configuration.txt  real time persistence  实时持久化  save on change  变化即保存  
FixedConfig.txt  constraints and output naming  约束与输出命名  DIDInfo DIDConfig uds and other filenames  DIDInfo DIDConfig uds 等文件名  

5 Detailed UI and interaction design  详细界面与交互设计

5.1 Left right rear domain LR_REAR  左右后域

5.1.1 Import testcase source  导入用例源  
UI field  测试用例 测试文件夹导入  
Actions  选择文件 选择文件夹 删除  
Result  选择后状态显示已选择并展示名称 删除后清空并恢复未选择  

5.1.2 Associate configuration tables  关联配置表  
The following fields are aligned with the current web page  以下字段与当前 Web 页面一致  

| UI field 界面字段 | Purpose 用途 | Generated outputs 生成 |
|---|---|---|
| Testcase file or folder  测试用例 测试文件夹导入 | data source for can scripts and xml tree  作为 can 与 xml 的数据源 | testmodule can and xml  生成到 testmodule 的 can 与 xml |
| IO_Mapping excel  IO_Mapping 配置表 | map physical signal to CAPL variable  物理信号与脚本变量映射 | no standalone output used in can generation  不单独生成文件 用于 can 生成 |
| DID_Config excel  DID_Config 配置表 | DID read write rules  DID 读写规则 | Configuration DIDConfig.txt  生成 DIDConfig.txt 到 Configuration |
| ResetDid_Value excel  ResetDid_Value 配置表 | reset default values and DID information source  reset 默认值与 DID 信息来源 | Configuration DIDInfo.txt  生成 DIDInfo.txt 到 Configuration |
| Clib excel  关键字集Clib 配置表 | keyword function mapping for cin and can  cin 与 can 的关键字函数映射 | testmodule cin and used by can  生成 cin 到 testmodule 并参与 can |
| UDS_ECU_qualifier  UDS_ECU_qualifier 下拉 | select ECU qualifier such as LDCU or RDCU  选择 ECU 标识 | Configuration uds.txt  生成 uds.txt 到 Configuration |

5.1.3 Filter and select cases  筛选与勾选用例  
Priority  用例优先级  checkbox group with select all invert  多选 支持全选反选  
Platform  测试平台  radio group single choice  单选  
Model  测试车型  radio group single choice  单选  
Target version  目标版本  checkbox group with select all invert  多选 支持全选反选  
Case selection tree  勾选用例树  file as parent sheet as child  父为文件 子为 sheet  
Tri state behavior  三态行为  parent reflects children selection  父节点随子节点显示全选 半选 未选  
Log level  日志级别  info warning error

6 Output deliverables and directory structure  输出成果物与目录结构

Output root is the selected Canoe project folder per tab  输出根目录为各页签选择的 Canoe 工程目录  
testmodule contains can xml cin  testmodule 包含 can xml cin  
Configuration contains DIDInfo DIDConfig uds and domain specific files  Configuration 包含 DIDInfo DIDConfig uds 与域相关文件  
log contains timestamped run logs  log 包含时间戳日志  

7 Error handling and logging  异常处理与日志

Missing required inputs should block run and keep status as not selected  缺少必填输入时阻止运行并保持未选择提示  
When run fails users locate details under log folder  运行失败时可在 log 目录查看详情
