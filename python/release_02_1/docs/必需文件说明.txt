一、重要提示
--------------------------------------------------------------------------------
1. 使用「打包后的 exe」时：必需配置文件放在 **exe 同级目录下的 config/** 中（即 exe 同目录/config/Configuration.ini、FixedConfig.ini）。
2. 使用「开发环境」时：必需配置文件放在**项目根目录下的 config/** 中（工程根 = 含主配置文件的目录）。
3. 所有配置文件建议使用 UTF-8 编码保存，避免中文路径或内容乱码。
4. 若缺少必需文件，工具可能无法启动或运行时报错；可选文件缺失时，对应功能不可用或使用默认值。


二、必需文件清单
--------------------------------------------------------------------------------

+------------------+------------------------------------------+------------------------------------------+
| 文件名           | 用途                                     | 缺失后果                                 |
+------------------+------------------------------------------+------------------------------------------+
| Configuration.ini| 主业务配置：输入/输出路径、IO 映射、      | 无法启动或无法找到输入/输出路径；        |
|                  | 关键字表、DID/CIN/UART/SOA 等各节配置。   | 各生成任务无法执行。                     |
+------------------+------------------------------------------+------------------------------------------+
| FixedConfig.ini  | 固定配置：部分输出文件名、路径、车型等。  | 使用代码内默认值；若默认不满足需求则     |
|                  | 与 Configuration 配合使用。               | 需在 Configuration 或界面中单独配置。    |
+------------------+------------------------------------------+------------------------------------------+
| filter_options.ini | 筛选项配置：等级、平台、车型、UDS_ECU   | 前端「等级/平台/车型」等下拉为空；       |
|                   | 等下拉列表内容，每类占一段。             | 用户仍可手动输入或使用 ALL。             |
+------------------+------------------------------------------+------------------------------------------+


三、推荐目录结构（与 exe 同目录，或开发时工程根目录）
--------------------------------------------------------------------------------

工程根目录/
│
├── app.py                             ← 开发时：主程序入口
├── build_exe.py                       ← 开发时：打包脚本
├── 测试用例生成工具_2026.x.x.exe      ← 打包后：可执行文件（与 app/build_exe 二选一使用）
│
├── config/                            ← 必需：配置文件目录
│   ├── Configuration.ini             ← 必需：主业务配置
│   ├── FixedConfig.ini               ← 固定配置（可选，缺失时走代码默认值）
│   └── filter_options.ini            ← 可选：筛选项（无则前端下拉链表为空；仅 .ini，代码无 .txt 名）
│
├── input/                             ← 推荐：存放输入 Excel 等
│   └── （根据 Configuration 中配置的路径放置用例表、IO 映射、关键字表、SOA 服务通信矩阵等）


四、主配置文件中的路径说明
--------------------------------------------------------------------------------
- 路径可使用「相对路径」或「绝对路径」。
- 相对路径：相对于「工程根」（打包后 = exe 所在目录，开发时 = 含 config 的项目根）解析，例如：./input/用例.xlsx、input/IO_mapping.xlsx。
- 绝对路径：直接写系统路径，如 D:/project/input/xxx.xlsx。
- 建议将 Excel、配置表等统一放在 input/ 下，在主配置文件中用相对路径引用，便于拷贝整个工程目录到其他机器。


五、编码与格式
--------------------------------------------------------------------------------
- 配置文件（`.ini`）：请使用 UTF-8 编码保存；若带 BOM 也可被识别。
- Excel（.xlsx/.xlsm）：由 openpyxl 读取，支持常见编码；表头与内容中的中文需与保存编码一致。


六、快速自检
--------------------------------------------------------------------------------
- 启动 exe 后若浏览器能打开界面，说明至少主配置文件已被找到（`Configuration.ini`）。
- 若「加载配置」或「开始运行」报错“未找到配置文件”“未配置 output_dir”等，请按上表检查对应文件是否存在、路径是否正确、编码是否为 UTF-8。
