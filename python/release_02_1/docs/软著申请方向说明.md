> 与《架构.txt》一致：本项目为 **Excel 驱动的通信测试用例生成工具链**。  
> **原则**：不把 CAN / CIN / XML / DID / UART 等输出格式拆成多个软著；约 **5～6 个** 方向，按功能模块划分。

## 使用方式

1. 打开本文件，选定要申请的条目（建议一次做满 5～6 条或按需删减）。

## 方向一：核心转译引擎

**全称建议**：`[公司]车载用例关键字CAPL转译软件 V1.0`

**对应代码内容**：`core/parser/`（步骤解析，`step_parser.py`） + `core/translator/`（`io_mapping`、`config_enum` 等）。

**为什么这么起名**：这里的“转译”指将 Excel 里的人写测试步骤（例如“Step Set Relay 1 ON”）转换为 CAPL 能识别的代码片段；该过程在计算机科学中可称为 Transpilation（转译）。

**基础信息（可直接填表）**：
- **硬件环境**：x86_64 工控机/办公 PC，8GB+ 内存，100GB+ 磁盘。  
- **软件环境**：Windows 10/11，Python 3.10+，openpyxl。  
- **编程语言**：Python。  
- **开发目的**：将人工编写测试步骤自动转为标准 CAPL 语义，降低脚本编写成本。  
- **面向领域/行业**：汽车电子测试、车载通信验证。  
- **软件主要功能**：本软件面向车载测试脚本自动化生产场景，提供从自然语言化测试步骤到可执行 CAPL 语义片段的完整转译能力。系统首先对 Excel 中的步骤文本进行结构化语法解析，识别动作动词、对象、通道、参数值、时序约束与前后依赖关系，并将口语化或项目约定写法映射为统一中间表达；在此基础上，软件内置关键字匹配与规则分派机制，能够根据不同测试意图自动选择对应转译路径，对常见别名、缩写、大小写差异、空白符噪声和多语言混写进行容错处理，保证解析稳定性。针对参数处理环节，系统支持数值单位归一、枚举值标准化、布尔语义校正、范围边界检查以及缺省值补全，避免人工脚本中因格式不一致导致的执行偏差；同时提供错误定位与诊断输出能力，可在步骤级、字段级给出失败原因、建议修正项和上下文片段，便于快速回溯原始输入。对于转译结果，软件支持按照模板化规则生成标准代码片段，并保留源步骤与目标语句的映射关系，支持审计追踪、二次复核和增量重生。系统还支持转译规则的版本化维护与灰度切换，可针对不同项目阶段启用差异化语义策略，保证历史脚本兼容与新规范落地并行推进；并可输出步骤覆盖度、规则命中率、异常分布等统计指标，为测试负责人评估脚本质量和持续优化规则库提供量化依据。通过“解析—匹配—标准化—转译—追踪”的流水线闭环，软件显著降低 CAPL 编写门槛，提升脚本一致性、可维护性与工程批量交付效率。  
- **软件的技术特点**：规则化转译链路、关键字映射可配置、异常定位可追溯。

## 方向二：多产物生成系统

**全称建议**：`[公司]多格式测试脚本与配置生成软件 V1.0`

**对应代码内容**：`generators/` 目录下所有子文件夹（`capl_can`、`capl_cin`、`capl_xml`、`capl_didconfig`、`capl_didinfo`、`capl_uart`）。

**为什么这么起名**：系统具备“多产物”输出能力。“测试脚本”对应 CAN、CIN、XML；“配置文件”对应 DID 与 UART 等内容，因此名称覆盖输出多格式能力。

**基础信息（可直接填表）**：
- **硬件环境**：x86_64 PC，8GB+ 内存，建议 SSD。  
- **软件环境**：Windows 10/11，Python 3.10+，Jinja2。  
- **编程语言**：Python。  
- **开发目的**：一次读取测试矩阵，批量生成多类型测试脚本与配置文件。  
- **面向领域/行业**：整车与零部件自动化测试、测试工程开发。  
- **软件主要功能**：本软件提供面向测试工程的一体化多产物生成能力，可在一次数据读取后并行输出 CAN、CIN、XML、DID 配置与 UART 相关文件，满足不同测试链路对脚本与配置成果物的协同需求。系统首先建立统一输入数据模型，对测试矩阵中的用例信息、信号定义、诊断项、通道属性与环境参数进行聚合与标准化，随后将结构化数据分发至各生成子模块，由各模块依据目标产物规范执行模板渲染、字段映射与格式校验。软件支持模板版本管理和生成规则配置，可根据项目差异切换命名策略、目录结构、头部注释、分片策略及编码格式，保证跨项目复用时输出仍符合本地规范；在生成过程中，系统对关键字段完整性、跨文件引用一致性、ID 冲突与依赖关系进行自动检查，并在发现异常时输出分层日志与可读错误说明，降低人工排查成本。产物输出层支持批量落盘、覆盖保护、增量更新与执行摘要汇总，可清晰记录每次生成的输入版本、模板版本与结果清单，便于追溯与复现。系统同时提供跨产物关联校验能力，可检查同一用例在不同目标文件中的名称、ID、通道和条件是否一致，防止多端脚本语义漂移；并可根据项目发布节奏执行分批生成与差异发布，仅输出变更范围内文件，减少重复构建时间与人工复核工作量。通过“统一入口、模块生成、规则渲染、校验落盘、结果归档”的方式，软件将分散的手工制作流程收敛为标准化生产流程，显著提升多格式脚本与配置交付的速度、稳定性与一致性。  
- **软件的技术特点**：统一生成入口、模块化渲染器、不同产物共享数据上下文。

## 方向三：工程配置中心

**全称建议**：`[公司]车载工程参数化配置与筛选软件 V1.0`

**对应代码内容**：`services/config_manager.py`（读写 `Configuration.ini`） + `core/case_filter.py`（等级、平台、车型等过滤逻辑）。

**基础信息（可直接填表）**：
- **硬件环境**：x86_64 PC，4GB+ 内存。  
- **软件环境**：Windows 10/11，Python 3.10+。  
- **编程语言**：Python。  
- **开发目的**：统一管理工程参数并实现用例准入控制，减少人工配置错误。  
- **面向领域/行业**：汽车测试项目管理、测试数据治理。  
- **软件主要功能**：本软件围绕车载测试工程的参数化管理与准入控制建立统一能力中心，首先提供工程级配置的集中维护功能，可对输入测试矩阵路径、输出目录、日志目录、模板目录、车型平台标识、执行阶段开关等关键参数进行新增、读取、更新与保存，确保不同项目在同一规则框架下快速初始化；其次提供配置合法性与依赖关系校验功能，针对路径可用性、必填项完整性、参数类型、枚举取值、互斥与联动关系进行自动检查，在任务启动前阻断无效配置进入执行链路；再次提供面向业务的用例筛选与准入判定能力，支持按平台、车型、等级、关键字、标签及状态等条件组合过滤，并对筛选结果进行数量统计、原因标注与可追溯记录，保证进入生成阶段的数据范围准确可控；同时具备运行态状态持久化能力，可保存最近一次工程上下文、筛选条件与运行标记，支持重复执行、回滚与快速复现；最后通过服务化接口向上层任务编排模块和可视化界面输出标准化配置数据，形成“配置输入—规则校验—准入筛选—状态固化—结果供给”的闭环流程。系统还支持多环境配置隔离与模板化初始化，可快速构建开发、联调、发布等不同运行档案，并提供配置变更留痕与对比能力，帮助团队识别关键参数改动带来的影响范围；在筛选执行侧可输出命中与剔除清单，关联原因标签与规则编号，便于项目评审与质量复盘。综合来看，本软件显著降低人工配置错误率，提升批量测试任务的一致性、可维护性与工程交付效率。  
- **软件的技术特点**：参数中心化、筛选规则可扩展、配置与业务逻辑解耦。

- **开发目的**：把多步骤生成任务标准化串联，并形成可审计日志链路。  
- **面向领域/行业**：自动化测试流程管理、工程交付追踪。  
- **软件主要功能**：本软件提供测试生成任务的全流程编排与日志管理能力，重点解决多步骤任务衔接复杂、失败定位困难与执行结果不可追溯的问题。系统首先基于可配置流程定义建立任务编排机制，可按依赖关系组织前置检查、数据解析、转译生成、产物写出与后处理动作，并支持串行与批处理模式下的统一调度；在执行前阶段，软件会自动完成输入完整性、环境可用性、配置合法性与路径冲突检测，提前拦截高风险运行条件，降低任务中断概率。执行中阶段提供状态机式运行控制，能够记录每个节点的开始时间、结束时间、耗时、输入输出摘要及成功失败标识，并支持异常捕获、重试策略与中断恢复，使长链路任务具备更强稳定性。日志管理方面，系统采用分层分类记录方式，分别保存业务日志、错误日志、诊断日志与汇总日志，支持按任务批次、模块名称、时间窗口与关键字检索，便于跨团队协同排查；同时对错误信息提供结构化归档，输出失败点、上下文参数与建议处理方向，提升问题闭环效率。软件还可生成运行报告与审计痕迹，确保每次生成过程具备可回放、可复现、可问责特性，并支持按批次对比不同执行结果的差异项，辅助定位回归问题与环境波动影响；在异常高发场景下可基于历史日志聚类形成问题画像，帮助团队优先治理高频故障链路。通过“编排控制—校验防护—稳定执行—分层日志—审计追踪”的体系化能力，本软件显著提升自动化测试任务的执行可靠性、维护效率与工程交付透明度。  
- **软件的技术特点**：流程状态机化、失败点精确定位、日志分层分类存储。
